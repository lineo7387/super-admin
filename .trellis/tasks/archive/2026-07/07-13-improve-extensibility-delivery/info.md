# Technical decisions

## Scope

本任务只实施架构与 starter 质量基线，不制作公开 demo 资产。

## Architecture

- Feature manifest is the single source of truth.
- Showcase routes are derived immutably through prefix/composition helpers.
- `packages/core` owns dependency-light manifest/layout metadata and pure helpers.
- App-local registries own Vue components and choose explicit neutral fallbacks.
- Unknown profile auth recipe and layout IDs must never masquerade as a branded built-in recipe.

## Vue component map

- `AppShell.vue`: shell orchestration only; resolve the selected layout registration, mount shared overlays, and wire workspace slots.
- `shell/layout-registry.ts`: app-local typed layout registrations, duplicate validation, preview presentation metadata, and the explicit neutral fallback.
- `shell/layouts/*Layout.vue`: presentational shell recipes with the shared `header-actions` and `workspace` slot contract.
- `GlobalPreferences.vue`: render selectable layout registrations; it does not own layout-ID branching.
- `StageWindowPreview.vue`: render route previews from registry presentation metadata; it does not infer chrome from layout IDs.
- `AuthLayout.vue`: stable auth root, shared preferences trigger, and active recipe orchestration only.
- `modules/auth/components/auth-recipe-registry*.ts`: pure registry contract plus generated installed-theme composition.
- `modules/auth/components/recipes/*AuthRecipe.vue`: high-fidelity presentational profile recipes receiving `eyebrow`, `title`, and `description`, with the form supplied through the default slot.
- `AuthRecipePanel.vue`: shared form-panel visual primitive used by every auth recipe.

## Starter quality model

- Default generation mode: `standard`.
- `--minimal` selects `minimal` quality mode.
- Standard output: ESLint, Vitest, lint/test/check scripts and representative tests.
- Minimal output: dev/typecheck/build/preview only; no quality-only config, test files, dependencies, or AI claims.
- Playwright remains repository/reference tooling for this task and is not generated by default.

## Compatibility

- Existing theme/charts/i18n flags remain valid.
- Existing `/examples/*` routes and compatibility redirects remain valid.
- Existing package runtime boundaries remain frontend-first and mock-backed.
- Changes that alter generated output require CLI tests, packed-starter validation, docs, and AI-context updates together.
